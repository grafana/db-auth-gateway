Usage of ./bin/db-auth-gateway:
  -auth.type string
    	Authentication type. Supported values: trust, forward_auth. When set to forward_auth, forward-auth.url is required (default "trust")
  -backend string
    	Determine if the backend is mimir, loki, pyroscope, or tempo (default "mimir")
  -client-ip-middleware.enabled
    	Flag used to enable the Client IP Middleware.
  -client-ip-middleware.extractor-type value
    	Specifies a comma-separated list of types of IP extractors to be used. Supported extractor types are "xff", "remoteaddr" and "xrealip".
  -client-ip-middleware.trusted-proxy-count int
    	Specifies the number of trusted proxies. The client IP will be the next to the left of the last trusted proxy IP. This has no impact if the "client-ip-middleware.extractor-type" is set to "remoteaddr" (default 2)
  -debug.block-profile-rate int
    	Fraction of goroutine blocking events that are reported in the blocking profile. 1 to include every blocking event in the profile, 0 to disable.
  -debug.mutex-profile-fraction int
    	Fraction at which mutex profile vents will be reported, 0 to disable
  -enable-h2c
    	Enable h2c.  This is required for HTTP2/GRPC without TLS.
  -forward-auth.cache-ttl duration
    	TTL for caching auth results (0 disables caching)
  -forward-auth.timeout duration
    	Timeout for requests to the external auth service (default 5s)
  -forward-auth.url string
    	URL of the external auth service (required when auth.type=forward_auth)
  -gateway.client-cluster-validation.label string
    	Primary cluster validation label.
  -gateway.connection-ttl.idle-check-frequency duration
    	The frequency of idle connections check. If during a check a connection is idle for at least -gateway.connection-ttl.max amount of time, its state is not kept anymore. Must be positive, and is taken into account only when -gateway.connection-ttl.max > 0. (default 1m0s)
  -gateway.connection-ttl.max duration
    	Maximum TTL of a TCP connection. When this limit is set, it will represent the maximum lifetime of TCP connections before the gateway sends a 'Connection: Close' response header to signal the client to close the connection. Set to <=0 to disable, or to at least -gateway.connection-ttl.min to enable.
  -gateway.connection-ttl.min duration
    	Minimum TTL of a TCP connection. When this limit is set, it will represent the minimum lifetime of TCP connections before the gateway sends a 'Connection: Close' response header to signal the client to close the connection. Set to <=0 to disable, or to at most -gateway.connection-ttl.max to enable.
  -gateway.dial.timeout duration
    	Timeout when dialing backend. This is the connection timeout. Set to 0 to disable. (default 5s)
  -gateway.distributor.bufferpool-size int
    	max number of byte buffers in the write buffer pool (default 100)
  -gateway.distributor.bufferpool-width int
    	capacity of byte array provided by the write buffer pool (default 1024)
  -gateway.distributor.downstream-duration-route-method-enabled
    	if enabled, the request_downstream_duration_seconds metric uses request route instead of request method as the method metrics label (applies to Mimir, Loki, and Pyroscope).
  -gateway.distributor.endpoint string
